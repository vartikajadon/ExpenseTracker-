AI-Powered Fintech Expense Tracker

🚀 Project Overview

The AI-Powered Fintech Expense Tracker is a full-stack web application designed to help users track their spending, manage budgets, and receive intelligent financial advice using Generative AI. It features a premium "Glassmorphism" UI, dynamic data visualization, and a secure backend proxy to protect sensitive data.

🛠 Technology Stack

Frontend (The User Interface)

HTML5 Semantic Elements
CSS3 Glassmorphism
Vanilla JavaScript (ES6+)
Chart.js Visualization

Backend & API

Node.js Runtime
Express.js Framework
Groq AI (Llama 3.3)
Dotenv (Security)

🏗 Core Architecture

1. Glassmorphism Design System

Characterized by translucent "frosted glass" elements. Implemented via background: rgba() for transparency and backdrop-filter: blur(14px) for the frosted effect. It creates a modern, premium aesthetic.

2. Secure Proxy Pattern

Crucial for fintech security. We never call the AI API directly from the frontend; instead, our Node.js server acts as a middleman, keeping the GROQ_API_KEY safely hidden in server environment variables.

3. State & Persistence

Leverages window.localStorage for client-side data persistence, ensuring user transactions survive page refreshes without needing a costly cloud database.

🎓 Interview Preparation

Q: Why did you use a Proxy server for the AI insights?

Security. If I call Groq directly from the frontend, my API key would be visible in the "Network" tab of the browser console. Using an Express proxy keeps the key safe on the server.

Q: How did you implement responsiveness for both phones and laptops?

I used a hybrid approach. CSS Grid for the modular components that stack into a single column on mobile, and clamp() functions for fluid typography that scales naturally with screen size.

Q: Explain the Glassmorphism implementation.

It relies on three layers: a translucent color, a high-blur filter, and a thin, high-contrast border to define the shape against the colorful background mesh.
