## 📄 Root Files

### 1. `index.html`
- **What is it?** The main entry point for the frontend user interface.
- **Why?** It defines the structural layout of the dashboard (cards, tables, forms) and serves as the container where JavaScript and CSS are injected to create the interactive experience.

### 2. `script.js`
- **What is it?** The "brain" of the frontend application.
- **Why?** It handles all user interactions, manages the expense data (state), renders dynamic charts using Chart.js, communicates with the backend server, and ensures data is saved to `localStorage`.

### 3. `style.css`
- **What is it?** The styling engine for the entire application.
- **Why?** It implements the premium **Glassmorphism** design system, handles the responsive grid layout (so the app looks great on phones and laptops), and manages the Light/Dark mode transitions.

### 4. `server.js`
- **What is it?** The Node.js/Express backend server.
- **Why?** Its most critical role is acting as a **Secure Proxy**. It keeps your private AI API keys hidden on the server so they aren't exposed in the browser. It also serves the frontend files when deployed.

### 5. `docs.html`
- **What is it?** A beautiful, standalone documentation page built directly into the app.
- **Why?** It provides a user-friendly way to understand the project's technical architecture and "Blueprint" without needing to read raw markdown files.

### 6. `package.json`
- **What is it?** The project's "ID card" and manifest.
- **Why?** It lists the project name, version, start scripts (e.g., `npm start`), and all the dependencies required for the server to run.

### 7. `package-lock.json`
- **What is it?** A detailed "snapshot" of the dependency tree.
- **Why?** It ensures that everyone who installs the project (and the deployment server) uses the exact same versions of libraries, preventing "it works on my machine" bugs.

### 8. `.env`
- **What is it?** A configuration file for environment variables (e.g., `GROQ_API_KEY`).
- **Why?** It separates sensitive credentials from the code. This file is **never** committed to public repositories (like GitHub) to prevent API key theft.

### 9. `.gitignore`
- **What is it?** A list of files that Git should ignore.
- **Why?** It prevents bulky or sensitive files (like `node_modules` or `.env`) from being uploaded to the cloud, keeping the repository clean and secure.
